## 项目简介

**Catan-Online** 是一款面向桌游爱好者、高校学生和休闲社交用户的在线卡坦岛网页游戏。

传统实体桌游存在玩家凑局困难、地图搭建繁琐、资源结算复杂、异地游玩不便等问题。本项目通过 Web 技术实现卡坦岛核心玩法的线上化，让玩家只需打开浏览器，即可创建房间并邀请好友实时对战。

项目采用前后端分离架构，基于 **Socket.io** 实现全双工实时通信，使用 **React + SVG** 渲染交互式六边形棋盘，并通过服务端权威校验机制保障多人对局状态一致性与公平性。

## 核心特性

### 在线多人联机

- 支持创建房间与房间号加入
- 支持玩家准备状态实时同步
- 支持多人实时对局
- 支持断线重连与房间状态恢复

### 完整卡坦岛核心玩法

- 随机生成标准卡坦岛地图
- 自动分配地形、点数与港口
- 支持初始布局阶段的蛇形放置顺序
- 支持掷骰子与资源自动分配
- 支持道路、村庄、城市建设
- 支持购买与使用发展卡
- 支持强盗移动、弃牌与抢夺资源
- 支持银行交易、港口交易与玩家间交易
- 支持最长道路、最大军队与胜利点结算
- 支持达到胜利条件后的自动结算

### 服务端权威校验

所有关键游戏逻辑均由服务端统一处理：

- 客户端只发送操作意图
- 服务端校验玩家身份、回合阶段与规则合法性
- 服务端统一更新游戏状态
- 服务端向所有玩家广播同步结果

该机制能够有效避免客户端篡改资源、伪造操作或产生状态冲突。

### 公私状态隔离

为防止玩家通过抓包或修改客户端代码获取他人手牌信息，系统实现了严格的状态脱敏机制：

- 服务端保存完整真实状态
- 广播给所有人的公开状态只包含资源数量、发展卡数量、胜利点等摘要信息
- 每位玩家仅能收到属于自己的真实资源与发展卡数据

### SVG 高精度棋盘渲染

项目使用 SVG 渲染卡坦岛六边形棋盘：

- 支持高清缩放
- 支持六边形地形纹理
- 支持顶点、边、港口的动态绘制
- 支持可建造位置高亮提示
- 支持交互式点击建设

### 全栈 TypeScript 与 Monorepo 架构

项目采用 `pnpm workspace` 管理前后端与共享模块：

```text
Catan_online
├── apps
│   ├── web        # 前端 React 应用
│   └── server     # 后端 Socket.io 服务
├── packages
│   └── shared     # 前后端共享类型与规则函数
├── package.json
└── pnpm-workspace.yaml
```

通过共享类型与纯函数逻辑，减少前后端重复代码，提高系统一致性与可维护性。

## 系统架构

本项目采用典型的 B/S 架构，整体分为四层：

1. **表现层 Web Client**
   - 基于 React 构建
   - 负责游戏大厅、房间页、棋盘、资源面板、弹窗等 UI 渲染
   - 捕获玩家点击、交易、建设等操作

2. **通信层 WebSocket**
   - 基于 Socket.io 实现
   - 负责房间事件、游戏指令、状态同步等实时通信

3. **业务逻辑层 Server**
   - 维护房间状态与游戏状态机
   - 校验所有玩家操作
   - 处理资源结算、强盗逻辑、发展卡逻辑、胜利判断等规则

4. **共享模型层 Shared**
   - 存放 TypeScript 类型定义
   - 存放资源校验、交易汇率、建筑成本等纯函数
   - 供前端和后端共同使用

核心通信流程如下：

```text
Client 发送 Action
        ↓
Server 校验身份、阶段与规则
        ↓
Server 修改权威游戏状态
        ↓
Server 生成公开状态与玩家私有状态
        ↓
Server 通过 STATE_SYNC 定向同步给客户端
        ↓
React 根据状态重新渲染界面
```

## 主要功能模块

### 1. 大厅与房间模块

- 创建房间
- 输入房间号加入房间
- 玩家列表展示
- 房主权限控制
- 玩家准备状态同步
- 开始游戏
- 断线重连提示

### 2. 棋盘生成模块

- 生成 19 块标准六边形地块
- 地形随机洗牌
- 沙漠地块处理
- 骰子点数分配
- 港口生成
- 顶点、边、地块拓扑关系构建

### 3. 游戏状态机模块

游戏流程被划分为多个阶段：

```text
lobby
  ↓
setup_settlement
  ↓
setup_road
  ↓
playing
  ↓
ended
```

同时支持多个子状态：

- 强盗状态 `robberInfo`
- 玩家交易状态 `tradeOffer`
- 道路建设卡状态 `roadBuildingInfo`
- 丰收之年卡状态 `yearOfPlentyPending`

### 4. 建设与规则校验模块

支持以下建设行为：

- 建造道路
- 建造村庄
- 升级城市
- 购买发展卡

校验内容包括：

- 当前是否为玩家回合
- 当前游戏阶段是否合法
- 玩家资源是否足够
- 顶点是否符合距离规则
- 道路是否与己方建筑或道路连通
- 是否存在其他玩家建筑阻断

### 5. 交易模块

支持多种交易方式：

- 与银行交易
- 与港口交易
- 玩家之间自由交易

系统会自动计算玩家当前可用的最优交易比例。

### 6. 强盗与弃牌模块

当玩家掷出 7 点时：

- 系统检测手牌超过上限的玩家
- 对应玩家弹出弃牌面板
- 当前玩家移动强盗
- 当前玩家可从相邻玩家处随机抢夺资源

### 7. 胜利结算模块

系统实时检测玩家胜利点：

- 村庄计分
- 城市计分
- 发展卡隐藏分
- 最长道路奖励
- 最大军队奖励

当玩家达到 10 分时，系统立即结束游戏并展示结算面板。

## 项目亮点

### 服务端权威架构

早期版本中，前端局部状态更新可能导致多客户端状态不一致。当前版本已重构为服务端权威架构，所有规则判断与状态修改均由后端完成，显著提升了同步稳定性与防作弊能力。

### 公私状态分离

不同于简单地广播完整游戏对象，本项目会针对每个玩家生成专属状态视图，避免泄露其他玩家的真实资源与发展卡信息。

### 图论建模棋盘

卡坦岛棋盘并非简单二维网格，而是由地块、顶点和边组成的复杂拓扑结构。本项目将棋盘抽象为无向图：

- `Hex` 表示资源地块
- `Vertex` 表示可建设村庄或城市的交叉点
- `Edge` 表示可建设道路的路径

这种设计便于进行道路连通性判断、相邻顶点校验和最长道路计算。

### 最长道路算法优化

最长道路计算本质上接近无向图最长简单路径问题。项目中采用 DFS，并结合以下优化：

- 构建玩家个人道路子图
- 剔除被其他玩家建筑阻断的节点
- 优先从奇数度节点开始搜索
- 使用剩余边数进行剪枝

在实际游戏规模下能够快速完成计算。

### SVG 动态棋盘

项目没有采用传统图片拼接方式，而是使用 SVG 绘制棋盘，使地图在不同分辨率下保持清晰，并且方便处理顶点、边、地块的交互事件。

### 可拖拽弹窗交互

交易、弃牌等操作弹窗可能遮挡棋盘。项目封装了拖拽 Hook，使关键弹窗可以自由移动，提升复杂桌游界面的可用性。

## 生产环境部署

### 1. 安装依赖

```bash
git clone https://github.com/SongJu15/Catan_online.git
cd Catan_online
pnpm install
```

### 2. 构建前端

```bash
pnpm -C apps/web run build
```

构建完成后，前端静态资源会输出到：

```text
apps/web/dist
```

### 3. 安装全局工具

```bash
npm install -g pm2 serve
```

### 4. 启动服务

启动后端服务：

```bash
pm2 start npx --name "catan-server" -- tsx apps/server/src/index.ts
```

启动前端静态服务：

```bash
pm2 start serve --name "catan-web" -- -s apps/web/dist -l 3000
```

> 注意：后端入口文件路径请根据实际项目结构确认。  
> 如果后端入口位于其他路径，请修改上述命令中的 `apps/server/src/index.ts`。

### 5. 保存 PM2 状态并设置开机自启

```bash
pm2 save
pm2 startup
```

### 6. 访问项目

在浏览器中访问：

```text
http://<服务器 IP>:3000
```

## 使用说明

### 创建房间

1. 打开游戏首页
2. 输入玩家昵称
3. 点击“创建房间”
4. 系统生成房间号
5. 将房间号分享给其他玩家

### 加入房间

1. 打开游戏首页
2. 输入玩家昵称
3. 输入好友分享的房间号
4. 点击“加入房间”

### 准备并开始游戏

1. 所有玩家进入房间后点击“准备”
2. 房主点击“开始游戏”
3. 系统自动生成地图并进入初始布局阶段

### 初始布局

玩家按照蛇形顺序依次放置：

- 2 个初始定居点
- 2 条初始道路

顺序示例：

```text
A → B → C → D → D → C → B → A
```

### 常规回合

每个玩家的回合通常包括：

1. 掷骰子
2. 系统自动分配资源
3. 交易资源
4. 建造道路、村庄或城市
5. 购买或使用发展卡
6. 结束回合

### 游戏结束

当任意玩家达到 10 个胜利点时，系统自动结束游戏并展示最终排名。

在生产环境构建后，项目具备较好的运行性能：

- 前端使用 Vite 进行压缩和 Tree-shaking
- 静态资源体积小，首屏加载速度快

在常规网络环境下，玩家操作到状态同步的端到端延迟可稳定控制在较低水平，能够满足实时桌游对交互流畅性的要求。

## 安全与公平性设计

本项目从架构层面考虑了多人桌游中的作弊风险：

- 客户端不保存完整真实房间状态
- 玩家无法直接修改自己的资源数量
- 玩家无法查看其他玩家的真实资源明细
- 所有建设、交易、发展卡、强盗等操作均由服务端校验
- 服务端会根据玩家身份、当前回合、游戏阶段和规则合法性决定是否接受操作

这种设计能够有效保障多人对局的公平性。
